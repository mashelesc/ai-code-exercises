Conditional logic simplifications

Five patterns applied to sales_report.py — before and after, with rationale.

validate_report_params set membership
before
if report_type not in ['summary', 'detailed', 'forecast']: raise ValueError(...) if output_format not in ['pdf', 'excel', 'html', 'json']: raise ValueError(...)
after
VALID_REPORT_TYPES = {"summary", "detailed", "forecast"} VALID_OUTPUT_FORMATS = {"pdf", "excel", "html", "json"} if report_type not in VALID_REPORT_TYPES: raise ValueError(...) if output_format not in VALID_OUTPUT_FORMATS: raise ValueError(...)
Valid values are defined once as module-level sets and reused in both validation and the dispatch table. Set membership is O(1) vs O(n) for list lookup, and adding a new format means editing one line in one place.
filter_sales_data unified branch
before
if isinstance(value, list): sales_data = [s for s in sales_data if s.get(key) in value] else: sales_data = [s for s in sales_data if s.get(key) == value]
after
allowed = (set(value) if isinstance(value, list) else {value}) result = [s for s in result if s.get(key) in allowed]
Wrapping a scalar in a set unifies both branches into one comprehension. The if/else now only decides how to build the allowed set — the filter itself is written once.
_compute_forecast list comprehension + divmod
before
growth_rates = [] for i in range(1, len(sorted_months)): prev = monthly_sales[sorted_months[i-1]] curr = monthly_sales[sorted_months[i]] if prev > 0: growth_rates.append( ((curr - prev) / prev) * 100) # month rollover for i in range(1, 4): month += 1 if month > 12: month = 1 year += 1
after
growth_rates = [ ((monthly_sales[sorted_months[i]] - monthly_sales[sorted_months[i-1]]) / monthly_sales[sorted_months[i-1]]) * 100 for i in range(1, len(sorted_months)) if monthly_sales[sorted_months[i-1]] > 0 ] # month rollover via divmod month_total, month = divmod(month, 12) month = month or 12 year += month_total if month != 12 else 0
The accumulator loop becomes a filtered list comprehension — the guard (if prev > 0) moves inline. Month rollover uses divmod to eliminate the manual if month > 12 branch entirely.
build_report_data dict.update pattern
before
if grouping: report_data['grouping'] = {...} if report_type == 'detailed': report_data['transactions'] = [...] if report_type == 'forecast': report_data['forecast'] = {...}
after
optional = [ {"grouping": {...}} if grouping else {}, {"transactions": [...]} if report_type == "detailed" else {}, {"forecast": {...}} if report_type == "forecast" else {}, ] for section in optional: report.update(section)
Optional keys are expressed declaratively as a list of dicts — empty dicts for absent sections, populated dicts otherwise. A single for/update loop merges them. Adding a new optional section is one list entry, not a new if block reaching into the report dict.
render_report dispatch table
before
if output_format == 'json': return report_data elif output_format == 'html': return _generate_html_report( report_data, include_charts) elif output_format == 'excel': return _generate_excel_report( report_data, include_charts) elif output_format == 'pdf': return _generate_pdf_report( report_data, include_charts)
after
_SERIALISERS = { "json": _identity, "html": _generate_html_report, "excel": _generate_excel_report, "pdf": _generate_pdf_report, } return _SERIALISERS[output_format](report_data)
A dict maps each valid format to its callable. Zero branching logic. Adding a new format is one dict entry. _identity makes the JSON pass-through explicit without a special-case branch, and the validated format key guarantees no KeyError at runtime.
